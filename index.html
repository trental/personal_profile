“paperclip” is a trading platform for bartering. If you’d like to trade your paperclip for a house, it might happen!

List items to trade and items wanted, and the application automatically connects an unlimited number of users for multi-person trades.
User connection is created with a recursive function - it searches for people that want what you have.

It does this by traversing the database as if it were a directed graph.
"paperclip" is a vision born out of a collaborative effort. I (Trent) created and coded the connection algorithm along with the rest of the back-end.

The "paperclip" tech stack includes:
- React
- NodeJS
- MongoDB
- Mongoose
- Express
paperclip features full CRUD on user items and user needs.

"statements" is a web application that compiles financial statements from a set of user entered transactions.
The PostGreSQL database is built with a bridge table to create a Many-to-Many relationship two data element tables.
"statements" is another of my collaborative projects. I was responsible for envisioning and building the back-end.

The "statements" tech stack includes:
- React
- Python3
- Django
- PostGreSQL
The app features full CRUD on user entries, along with detail for each transaction's individual impact the three financial statements.
